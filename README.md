## 🎯 Core Philosophy
Outrivia is built on three pillars:
1.  **Autonomy**: The system handles the "grunt work" of prospecting and data entry autonomously.
2.  **Intelligence**: Every message is context-aware, generated by Mistral AI using deep-scraped profile data.
3.  **Real-time Visibility**: Users are never in the dark. High-fidelity logs are streamed live as the system processes data.

---

## ⚡ The Autonomous Pipeline
Outrivia's core strength lies in its 8-stage intelligent pipeline:

| Stage | Action | Description |
| :--- | :--- | :--- |
| **1. Intent Parsing** | AI Analysis | Parses natural language goals (e.g., *"Find AI founders in SF"*) into structured search parameters. |
| **2. Discovery** | SerpAPI Search | Dynamically generates and executes Google/LinkedIn search queries. |
| **3. Deduplication** | Database Filter | Cross-references URLs with Supabase to ensure no prospect is ever processed twice. |
| **4. Enrichment** | Apify Scraping | High-fidelity scraping of LinkedIn profiles (experience, skills, company details). |
| **5. Email Extraction**| Data Validation | Extracts verified professional email addresses directly from enriched results. |
| **6. AI Generation** | Mistral AI | Generates hyper-personalized messages based on specific prospect history and skills. |
| **7. Approval Flow** | Human Review | Presents generated drafts in a clean UI for user review, editing, and approval. |
| **8. Delivery** | Resend API | Sends approved emails via a high-deliverability infrastructure. |

## 🛠️ Technical Architecture

### 🛡️ Backend (FastAPI)
Built for high concurrency and low latency:
- **Asynchronous Execution**: Uses FastAPI's `BackgroundTasks` for non-blocking pipeline execution.
- **WebSocket Gateway**: A custom `ConnectionManager` streams live pipeline logs directly to the dashboard.
- **Service Modularization**: Decoupled service layers for AI, Search, Scraping, and Email.

### 🎨 Frontend (React + Vite)
Built for a premium user experience:
- **Reactive State**: Real-time dashboard updates via WebSockets.
- **Tailwind CSS & shadcn/ui**: Modern, accessible, and high-quality UI components.
- **TypeScript**: Full type safety across the entire application.

### 🗄️ Database (Supabase)
Used for state management and persistence:
- **`goals`**: Tracks user intents and their processing status.
- **`prospects`**: Stores enriched profile data with a unique constraint on `linkedin_url`.
- **`messages`**: Manages outreach drafts, approval states, and delivery status.
- **`outreach_logs`**: Persists pipeline events for auditing and debugging.

## 🖇️ API Reference

### Core Endpoints
- `POST /goal`: Initiates the entire autonomous pipeline. Returns a `job_id`.
- `GET /prospects/{job_id}`: Retrieves all prospects discovered for a specific goal.
- `GET /messages/{job_id}`: Retrieves all AI-generated outreach drafts.
- `POST /approve`: Receives a list of prospect IDs to mark for delivery.
- `POST /send`: Triggers the final email delivery process for approved prospects.

### Real-time Streaming
- `WS /logs/{job_id}`: Establishes a WebSocket connection to receive real-time status updates (JSON).

## 🚀 Environment Setup
Outrivia requires several API keys to function in production mode. Create a `.env` file in the `backend/` directory:

```bash
# Supabase
SUPABASE_URL=https://your-project.supabase.co
SUPABASE_KEY=your-anon-key

# AI & Discovery
MISTRAL_API_KEY=your-mistral-key
SERPAPI_API_KEY=your-serpapi-key
APIFY_API_KEY=your-apify-token

# Delivery
RESEND_API_KEY=your-resend-key
FROM_EMAIL=onboarding@resend.dev
```

## ⚡ Installation & Performance

### Backend Deployment
1.  **Virtual Env**: `python3 -m venv venv && source venv/bin/activate`
2.  **Dependencies**: `pip install -r requirements.txt`
3.  **Run Server**: `python main.py`

### Frontend Deployment
1.  **Install**: `npm install`
2.  **Dev Mode**: `npm run dev`

### Performance Metrics
- **Pipeline Start Latency**: < 200ms
- **Intent Extraction**: ~1-2s (Mistral Large)
- **Live Log Latency**: ~50ms (WebSocket)
